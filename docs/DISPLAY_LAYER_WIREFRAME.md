# 展示页低保真线框与信息架构

## 1. 页面目标

本页用于承接 OpenClaw 音乐技能的可视化反馈，重点展示：

1. 当前播放
2. 歌词
3. 下载任务
4. 最近动作反馈

## 2. 信息架构

### 2.1 一级模块

1. 顶部状态栏
2. 当前播放卡片
3. 歌词主区域
4. 下载任务区
5. 最近动作反馈区

### 2.2 数据优先级

优先级从高到低：

1. 当前播放状态
2. 当前歌曲信息
3. 歌词内容
4. 下载进度
5. 最近反馈消息

### 2.3 页面数据来源

1. 当前播放：会话上下文 + 播放状态
2. 歌词：歌词接口或本地歌词源
3. 下载任务：下载任务仓储
4. 动作反馈：最近一次操作结果

## 4. 交互分区说明

### 4.1 顶部状态栏

作用：

1. 标识当前页面
2. 提供全局状态提示
3. 提供最小窗口操作

展示内容：

1. 面板标题
2. 当前连接状态
3. 收起按钮

### 4.2 当前播放卡片

作用：

1. 用户第一眼确认正在播放什么
2. 集中展示最高频操作

展示内容：

1. 封面占位
2. 歌名
3. 歌手
4. 专辑
5. 当前播放状态
6. 播放进度
7. 高频操作按钮

### 4.3 歌词区

作用：

1. 承接持续阅读内容
2. 在没有歌词时提供稳定空状态

空状态：

1. `当前歌曲暂无歌词`
2. `歌词加载中`
3. `未匹配到歌词源`

### 4.4 下载任务区

作用：

1. 替代用户反复询问“下载好了没”
2. 让下载从一次性消息变成持续可见状态

展示内容：

1. 最近任务名
2. 状态
3. 进度条
4. 文件路径

### 4.5 最近反馈区

作用：

1. 展示最近一次成功或失败动作
2. 提供短时状态确认

示例：

1. `已收藏《录音》`
2. `已把《录音》加入歌单“学习”`
3. `下载完成`

## 5. 页面尺寸建议

推荐 MVP 尺寸：

1. 宽度：420px 到 520px
2. 高度：720px 到 860px

适合：

1. OpenClaw 侧边页
2. 桌面悬浮窗

## 6. 移动层级

视觉层级建议：

1. 当前播放卡片最高
2. 歌词区第二
3. 下载任务区第三
4. 最近反馈区最低但固定可见

## 7. 原型实现建议

第一版前端原型只需要：

1. 静态布局
2. 基础样式
3. 从本地接口轮询状态
4. 用占位歌词和下载数据驱动渲染

不需要先做：

1. 完整播放器动画
2. 拖拽调整
3. 复杂主题切换
