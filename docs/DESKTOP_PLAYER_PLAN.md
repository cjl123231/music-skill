# 小乐桌面播放器方案

## 新定位

不再把 `panel` 当成主产品。

新的主产品是：

- 小乐桌面播放器
- 本地播放器后端
- XiaoLe Agent

Web 页只保留为桌面壳中的渲染层，不再让用户自己开浏览器访问。

## 桌面端结构

### 1. 桌面前端

- Electron 主进程负责窗口、托盘、生命周期
- Electron 渲染层负责播放器 UI

当前已落地：

- `desktop/main.mjs`
- `desktop/preload.cjs`

### 2. 播放器后端

- 当前沿用本地 HTTP 后端
- 由桌面主进程自动拉起
- 默认端口：`3330`

### 3. XiaoLe Agent

- 继续保留当前 agent 逻辑
- 所有音乐命令仍统一进 XiaoLe

## 当前实现方式

桌面播放器启动后会：

1. 检查本地后端是否已在 `3330` 运行
2. 没运行就自动启动 `src/interfaces/http/server-runner.ts`
3. Electron 窗口加载内部播放器页面

这样用户体验从：

- 打开浏览器访问 panel

改成：

- 直接打开一个独立播放器窗口

## 运行命令

```powershell
pnpm desktop:start
```
